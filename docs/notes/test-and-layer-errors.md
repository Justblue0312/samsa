## 1. Testing Philosophy

Each layer has a single job. Tests should verify that job and nothing else.

```
repository   →  does it read/write the database correctly?
usecase      →  does it enforce business rules correctly?
handler      →  does it handle HTTP concerns correctly?
```

**The golden rule: mock the boundary below you, never the layer you are testing.**

```
handler test    →  mock Usecase interface
usecase test    →  mock Repository + local cross-feature interfaces
repository test →  real database (testcontainers)
```

Never mock the database in a usecase test. Never test business rules in a handler test. If a test is hard to write, it is usually a sign that the layer is doing too much.

## 2. Test File Location

**Tests live next to the code they test — never in a subfolder.**

```
feature/user/
  usecase_impl.go
  usecase_impl_test.go    ✅ — next to the code

feature/user/tests/
  usecase_impl_test.go    ❌ — never do this
```

Go's test tooling is built around this convention. Moving tests into a subfolder means:

- `go test ./internal/feature/user/` no longer finds your tests
- `package user_test` cannot test the public API of the `user` package from a different path
- Every CI command needs explicit subfolder paths

The **only** exception is cross-feature tests that genuinely span multiple packages — those live in a root-level `tests/` folder:

```
tests/
  integration/
    user_project_flow_test.go   ← spans user + project features
  e2e/
    auth_flow_test.go
```

Always use `package user_test` (external test package), never `package user`. This forces testing through the public API and prevents coupling to unexported internals.

```go
// ✅ external test package
package user_test

// ❌ internal — can access unexported fields, creates tight coupling
package user
```


## 4. Shared Test Infrastructure (`testkit`)

All shared test infrastructure lives in `internal/testkit`. This is the single place that knows how to spin up containers, run migrations, provide transaction isolation, and create test data.

### Why `testkit` Exists

Without it, every feature package duplicates container setup, migration logic, and seed helpers. With it, each test file is three lines of setup.

### Part A — Shared Database Connection

The container starts **once per test binary** using `sync.Once`, not once per file or per package. This keeps the full test suite fast.

```go
// internal/testkit/db.go
package testkit

import (
    "context"
    "sync"
    "testing"

    "github.com/jackc/pgx/v5"
    "github.com/jackc/pgx/v5/pgxpool"
    tcpostgres "github.com/testcontainers/testcontainers-go/modules/postgres"
)

var (
    sharedPool     *pgxpool.Pool
    sharedPoolOnce sync.Once
    sharedPoolErr  error
)

// NewDB returns a shared pool — the container starts once per test binary.
// Each test gets isolation via NewTx, not via separate containers.
func NewDB(t *testing.T) *pgxpool.Pool {
    t.Helper()

    sharedPoolOnce.Do(func() {
        ctx := context.Background()

        container, err := tcpostgres.RunContainer(ctx,
            tcpostgres.WithDatabase("testdb"),
            tcpostgres.WithUsername("test"),
            tcpostgres.WithPassword("test"),
        )
        if err != nil {
            sharedPoolErr = err
            return
        }

        connStr, err := container.ConnectionString(ctx, "sslmode=disable")
        if err != nil {
            sharedPoolErr = err
            return
        }

        pool, err := pgxpool.New(ctx, connStr)
        if err != nil {
            sharedPoolErr = err
            return
        }

        if err := RunMigrations(pool); err != nil {
            sharedPoolErr = err
            return
        }

        sharedPool = pool
    })

    if sharedPoolErr != nil {
        t.Fatalf("testkit: failed to initialise test DB: %v", sharedPoolErr)
    }

    return sharedPool
}
```

### Part B — Transaction Isolation

Each test wraps all its DB operations in a **transaction that always rolls back**. This means:

- Tests never see each other's data
- No cleanup functions needed — rollback handles everything
- Tests can run in parallel safely
- No fragile DELETE statements in teardown

```go
// internal/testkit/db.go (continued)

// NewTx starts a transaction for the duration of a single test.
// The transaction is ALWAYS rolled back via t.Cleanup — even if the test panics.
// Pass the returned tx into your repository constructor instead of the pool.
func NewTx(t *testing.T, pool *pgxpool.Pool) pgx.Tx {
    t.Helper()

    tx, err := pool.Begin(context.Background())
    if err != nil {
        t.Fatalf("testkit: failed to begin tx: %v", err)
    }

    t.Cleanup(func() {
        _ = tx.Rollback(context.Background())
    })

    return tx
}
```

### Part C — Migrations

```go
// internal/testkit/migrate.go
package testkit

import (
    "github.com/jackc/pgx/v5/pgxpool"
    "github.com/golang-migrate/migrate/v4"
    pgxv5 "github.com/golang-migrate/migrate/v4/database/pgx/v5"
)

func RunMigrations(pool *pgxpool.Pool) error {
    driver, err := pgxv5.WithInstance(pool, &pgxv5.Config{})
    if err != nil {
        return err
    }
    m, err := migrate.NewWithDatabaseInstance("file://../../../db/migrations", "pgx5", driver)
    if err != nil {
        return err
    }
    if err := m.Up(); err != nil && err != migrate.ErrNoChange {
        return err
    }
    return nil
}
```

### Part D — Test Data Factories

Factories create valid records with sensible defaults. Tests only specify fields they care about.

```go
// internal/testkit/factory/user.go
package factory

import (
    "context"
    "testing"

    "github.com/jackc/pgx/v5"
    "github.com/stretchr/testify/require"

    "myapp/internal/feature/user"
)

type UserOpts struct {
    ID    string
    Name  string
    Email string
    Role  user.Role
}

// User inserts a user into the DB within the given transaction and returns the model.
// Any zero-value field gets a sensible default — test only sets what it cares about.
func User(t *testing.T, tx pgx.Tx, opts UserOpts) *user.User {
    t.Helper()

    if opts.ID    == "" { opts.ID    = "usr_" + randID() }
    if opts.Name  == "" { opts.Name  = "Test User" }
    if opts.Email == "" { opts.Email = opts.ID + "@example.com" }
    if opts.Role  == "" { opts.Role  = user.RoleMember }

    _, err := tx.Exec(context.Background(),
        `INSERT INTO users (id, name, email, role) VALUES ($1, $2, $3, $4)`,
        opts.ID, opts.Name, opts.Email, opts.Role,
    )
    require.NoError(t, err)

    return &user.User{
        ID:    opts.ID,
        Name:  opts.Name,
        Email: opts.Email,
        Role:  opts.Role,
    }
}
```

```go
// internal/testkit/factory/project.go
package factory

type ProjectOpts struct {
    ID      string
    Name    string
    OwnerID string
}

func Project(t *testing.T, tx pgx.Tx, opts ProjectOpts) *project.Project {
    t.Helper()

    if opts.ID      == "" { opts.ID      = "proj_" + randID() }
    if opts.Name    == "" { opts.Name    = "Test Project" }
    if opts.OwnerID == "" { opts.OwnerID = User(t, tx, UserOpts{}).ID } // auto-create owner

    _, err := tx.Exec(context.Background(),
        `INSERT INTO projects (id, name, owner_id) VALUES ($1, $2, $3)`,
        opts.ID, opts.Name, opts.OwnerID,
    )
    require.NoError(t, err)

    return &project.Project{ID: opts.ID, Name: opts.Name, OwnerID: opts.OwnerID}
}
```

```go
// internal/testkit/factory/helpers.go
package factory

import (
    "crypto/rand"
    "encoding/hex"
)

func randID() string {
    b := make([]byte, 4)
    _, _ = rand.Read(b)
    return hex.EncodeToString(b)
}
```

### Part E — `TestMain` for Package-Level Setup

Use `TestMain` when a package needs something initialised **once before any test runs**.

```go
// feature/user/main_test.go
package user_test

import (
    "os"
    "testing"

    "myapp/internal/testkit"
)

// TestMain runs once for the entire package test suite.
// Warms up the shared DB container so the first test does not pay the startup cost.
func TestMain(m *testing.M) {
    testkit.NewDB(nil) // nil is safe here — errors are fatal inside NewDB
    os.Exit(m.Run())
}
```

## 5. Repository Testing

Repository tests run against a **real database** via `testkit`. Each test gets its own transaction that rolls back automatically — no cleanup code needed.

```go
// feature/user/repository_impl_test.go
package user_test

import (
    "context"
    "testing"

    "github.com/stretchr/testify/assert"
    "github.com/stretchr/testify/require"

    "myapp/internal/feature/user"
    "myapp/internal/testkit"
    "myapp/internal/testkit/factory"
    "myapp/gen/sqlc"
)

func TestRepository_FindByID(t *testing.T) {
    pool := testkit.NewDB(t)

    t.Run("returns user when found", func(t *testing.T) {
        tx   := testkit.NewTx(t, pool)          // isolated — rolls back after this subtest
        repo := user.NewRepository(sqlc.New(tx)) // repo uses tx, not pool directly

        seeded := factory.User(t, tx, factory.UserOpts{
            ID:    "usr_1",
            Email: "alice@example.com",
        })

        got, err := repo.FindByID(context.Background(), seeded.ID)

        require.NoError(t, err)
        assert.Equal(t, seeded.ID, got.ID)
        assert.Equal(t, seeded.Email, got.Email)
        // tx rolls back here automatically — no cleanup needed
    })

    t.Run("returns ErrNotFound when missing", func(t *testing.T) {
        tx   := testkit.NewTx(t, pool)
        repo := user.NewRepository(sqlc.New(tx))

        _, err := repo.FindByID(context.Background(), "does_not_exist")

        assert.ErrorIs(t, err, user.ErrNotFound)
    })
}

func TestRepository_Save(t *testing.T) {
    pool := testkit.NewDB(t)

    t.Run("saves user successfully", func(t *testing.T) {
        tx   := testkit.NewTx(t, pool)
        repo := user.NewRepository(sqlc.New(tx))

        err := repo.Save(context.Background(), &user.User{
            ID:    "usr_new",
            Name:  "Bob",
            Email: "bob@example.com",
            Role:  user.RoleMember,
        })
        require.NoError(t, err)

        got, err := repo.FindByID(context.Background(), "usr_new")
        require.NoError(t, err)
        assert.Equal(t, "Bob", got.Name)
    })

    t.Run("returns ErrEmailTaken on duplicate email", func(t *testing.T) {
        tx   := testkit.NewTx(t, pool)
        repo := user.NewRepository(sqlc.New(tx))

        factory.User(t, tx, factory.UserOpts{Email: "taken@example.com"})

        err := repo.Save(context.Background(), &user.User{
            ID:    "usr_dup",
            Email: "taken@example.com",
        })

        assert.ErrorIs(t, err, user.ErrEmailTaken)
    })
}
```

### What to Test in Repository

- Correct model returned on success
- `ErrNotFound` returned on miss
- Unique constraint translated to domain error (`ErrEmailTaken`)
- Update only changes the expected fields
- Delete removes the row and subsequent FindByID returns `ErrNotFound`

## 6. Usecase Testing

Usecase tests are **pure unit tests** — no DB, no containers, no network. All interfaces are mocked. Should run in milliseconds. Use table-driven tests to cover all business rule branches.

```go
// feature/user/usecase_impl_test.go
package user_test

import (
    "context"
    "errors"
    "testing"

    "github.com/stretchr/testify/assert"
    "github.com/stretchr/testify/mock"
    "github.com/stretchr/testify/require"

    "myapp/internal/feature/user"
    "myapp/internal/feature/user/mocks"
)

func TestUsecase_CreateUser(t *testing.T) {
    ctx := context.Background()

    tests := []struct {
        name       string
        input      user.CreateUserInput
        setupMocks func(repo *mocks.Repository)
        wantErr    error
        wantName   string
    }{
        {
            name:  "creates user successfully",
            input: user.CreateUserInput{Name: "Alice", Email: "alice@example.com", Role: user.RoleMember},
            setupMocks: func(repo *mocks.Repository) {
                repo.On("FindByEmail", ctx, "alice@example.com").Return(nil, user.ErrNotFound)
                repo.On("Save", ctx, mock.AnythingOfType("*user.User")).Return(nil)
            },
            wantName: "Alice",
        },
        {
            name:  "returns ErrEmailTaken when email exists",
            input: user.CreateUserInput{Name: "Alice", Email: "taken@example.com"},
            setupMocks: func(repo *mocks.Repository) {
                repo.On("FindByEmail", ctx, "taken@example.com").
                    Return(&user.User{ID: "existing"}, nil)
            },
            wantErr: user.ErrEmailTaken,
        },
        {
            name:  "returns ValidationError on empty name",
            input: user.CreateUserInput{Name: "", Email: "alice@example.com"},
            setupMocks: func(repo *mocks.Repository) {
                // repo must not be called when input is invalid
            },
            wantErr: &user.ValidationError{Field: "name"},
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            repo := mocks.NewRepository(t)
            sc   := mocks.NewSessionChecker(t)
            tt.setupMocks(repo)

            uc  := user.NewUsecase(repo, sc, nil)
            got, err := uc.CreateUser(ctx, tt.input)

            if tt.wantErr != nil {
                require.Error(t, err)
                var valErr *user.ValidationError
                if errors.As(tt.wantErr, &valErr) {
                    var gotErr *user.ValidationError
                    assert.ErrorAs(t, err, &gotErr)
                    assert.Equal(t, valErr.Field, gotErr.Field)
                } else {
                    assert.ErrorIs(t, err, tt.wantErr)
                }
                return
            }

            require.NoError(t, err)
            assert.Equal(t, tt.wantName, got.Name)
            repo.AssertExpectations(t)
        })
    }
}

func TestUsecase_DeleteUser(t *testing.T) {
    ctx := context.Background()

    t.Run("blocks delete when active session exists", func(t *testing.T) {
        repo := mocks.NewRepository(t)
        sc   := mocks.NewSessionChecker(t)

        sc.On("HasActiveSession", ctx, "usr_1").Return(true, nil)

        uc  := user.NewUsecase(repo, sc, nil)
        err := uc.DeleteUser(ctx, "usr_1")

        assert.ErrorIs(t, err, user.ErrHasActiveSession)
        repo.AssertNotCalled(t, "Delete") // verify side effect did NOT happen
    })

    t.Run("deletes when no active session", func(t *testing.T) {
        repo := mocks.NewRepository(t)
        sc   := mocks.NewSessionChecker(t)

        sc.On("HasActiveSession", ctx, "usr_1").Return(false, nil)
        repo.On("Delete", ctx, "usr_1").Return(nil)

        uc  := user.NewUsecase(repo, sc, nil)
        err := uc.DeleteUser(ctx, "usr_1")

        require.NoError(t, err)
        repo.AssertExpectations(t)
    })
}
```

### What to Test in Usecase

- Happy path returns correct result
- Each business rule that can block an operation
- Repo methods that must NOT be called on early returns — use `AssertNotCalled`
- Side effects (task dispatch) only triggered on success
- Each error case from the repo is handled or bubbled correctly

## 7. Handler Testing

Handler tests verify **HTTP concerns only** — binding, validation, RBAC, status codes, response shape. Usecase is mocked and returns controlled values.

```go
// feature/user/handler_http_test.go
package user_test

import (
    "bytes"
    "encoding/json"
    "net/http"
    "net/http/httptest"
    "testing"

    "github.com/gin-gonic/gin"
    "github.com/stretchr/testify/assert"
    "github.com/stretchr/testify/mock"
    "github.com/stretchr/testify/require"

    "myapp/internal/feature/user"
    "myapp/internal/feature/user/mocks"
)

func TestHTTPHandler_CreateUser(t *testing.T) {
    gin.SetMode(gin.TestMode)

    tests := []struct {
        name       string
        body       map[string]any
        callerRole string
        setupMocks func(uc *mocks.Usecase)
        wantStatus int
        wantCode   string
    }{
        {
            name:       "201 on valid request by admin",
            body:       map[string]any{"name": "Alice", "email": "alice@example.com", "role": "member"},
            callerRole: "admin",
            setupMocks: func(uc *mocks.Usecase) {
                uc.On("CreateUser", mock.Anything, user.CreateUserInput{
                    Name: "Alice", Email: "alice@example.com", Role: user.RoleMember,
                }).Return(&user.User{ID: "usr_1", Name: "Alice"}, nil)
            },
            wantStatus: http.StatusCreated,
        },
        {
            name:       "400 when email is missing",
            body:       map[string]any{"name": "Alice"},
            callerRole: "admin",
            setupMocks: func(uc *mocks.Usecase) {},
            wantStatus: http.StatusBadRequest,
            wantCode:   "BAD_REQUEST",
        },
        {
            name:       "403 when caller is not admin",
            body:       map[string]any{"name": "Alice", "email": "alice@example.com", "role": "member"},
            callerRole: "member",
            setupMocks: func(uc *mocks.Usecase) {},
            wantStatus: http.StatusForbidden,
            wantCode:   "FORBIDDEN",
        },
        {
            name:       "409 when email is taken",
            body:       map[string]any{"name": "Alice", "email": "taken@example.com", "role": "member"},
            callerRole: "admin",
            setupMocks: func(uc *mocks.Usecase) {
                uc.On("CreateUser", mock.Anything, mock.Anything).
                    Return(nil, user.ErrEmailTaken)
            },
            wantStatus: http.StatusConflict,
            wantCode:   "CONFLICT",
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            uc := mocks.NewUsecase(t)
            tt.setupMocks(uc)

            w, c := newTestContext(t, http.MethodPost, jsonBody(t, tt.body))
            c.Set("role", tt.callerRole)

            user.NewHTTPHandler(uc).CreateUser(c)

            assert.Equal(t, tt.wantStatus, w.Code)
            if tt.wantCode != "" {
                var resp map[string]any
                require.NoError(t, json.Unmarshal(w.Body.Bytes(), &resp))
                assert.Equal(t, tt.wantCode, resp["code"])
            }
            uc.AssertExpectations(t)
        })
    }
}

// ── shared HTTP test helpers ──────────────────────────────────────────────────

func newTestContext(t *testing.T, method string, b []byte) (*httptest.ResponseRecorder, *gin.Context) {
    t.Helper()
    w := httptest.NewRecorder()
    c, _ := gin.CreateTestContext(w)
    c.Request = httptest.NewRequest(method, "/", bytes.NewBuffer(b))
    c.Request.Header.Set("Content-Type", "application/json")
    return w, c
}

func jsonBody(t *testing.T, v map[string]any) []byte {
    t.Helper()
    b, err := json.Marshal(v)
    require.NoError(t, err)
    return b
}
```

### What to Test in Handler

- Valid request returns correct status and response shape
- Missing or invalid fields return 400
- Insufficient role returns 403
- Each domain error maps to the correct HTTP status
- Usecase is **not called** when validation or RBAC blocks early

## 8. Mock Generation & Organization

### Location — Per Feature, Not Centralized

```
feature/user/mocks/    ✅ close to what it mocks, independent lifecycle
feature/auth/mocks/    ✅

internal/mocks/        ❌ must import every feature — recreates cycle problems
```

### Generation With mockery

Add `//go:generate` above each interface:

```go
// feature/user/repository.go
//go:generate mockery --name=Repository --output=./mocks --outpkg=mocks
//go:generate mockery --name=sessionChecker --output=./mocks --outpkg=mocks

package user
```

```go
// feature/user/usecase.go
//go:generate mockery --name=Usecase --output=./mocks --outpkg=mocks

package user
```

Regenerate everything in one command:

```bash
go generate ./...
```

Add `.mockery.yaml` at project root for consistent config:

```yaml
# .mockery.yaml
with-expecter: true
mockname: "{{.InterfaceName}}"
outpkg: mocks
```

### Rules for Mocks

- Never edit generated files by hand
- Commit generated mocks — keeps CI fast, no generation step needed
- Regenerate whenever an interface signature changes

## 9. Error Handling Philosophy

There are four distinct error layers, each with a specific responsibility:

```
domain errors  →  express what went wrong in business terms
repository     →  translate storage errors into domain errors
usecase        →  enforce rules, wrap unexpected errors with context
handler        →  translate domain errors into HTTP responses
```

**The fundamental rules:**

1. Storage errors (`sql.ErrNoRows`, postgres codes) never escape the repository
2. HTTP status codes never appear below the handler
3. `fmt.Errorf("context: %w", err)` wraps unexpected errors with location
4. The default handler case always logs and returns 500 — never expose internal detail

---

## 10. Domain Errors

Domain errors live in `model.go`. They are the shared vocabulary of the entire stack.

### Sentinel Errors — For Known Business Outcomes

Use when the caller only needs to know **what** went wrong. Compare with `errors.Is()`.

```go
// feature/user/model.go
package user

import "errors"

var (
    ErrNotFound         = errors.New("user not found")
    ErrEmailTaken       = errors.New("email already taken")
    ErrUnauthorized     = errors.New("unauthorized")
    ErrHasActiveSession = errors.New("user has an active session")
)
```

### Typed Errors — When the Caller Needs Detail

Use when the caller needs to **inspect fields**. Compare with `errors.As()`.

```go
// feature/user/model.go

type ValidationError struct {
    Field   string
    Message string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("validation failed on field '%s': %s", e.Field, e.Message)
}

type NotFoundError struct {
    ID string
}

func (e *NotFoundError) Error() string {
    return fmt.Sprintf("user '%s' not found", e.ID)
}
```

### When to Use Which

| Situation                | Type                         | Compared With          |
| ------------------------ | ---------------------------- | ---------------------- |
| Resource not found       | `*NotFoundError`             | `errors.As`            |
| Unique constraint        | `ErrEmailTaken`              | `errors.Is`            |
| Business rule blocked    | `ErrHasActiveSession`        | `errors.Is`            |
| Input validation failed  | `*ValidationError`           | `errors.As`            |
| Unexpected / infra error | `fmt.Errorf("ctx: %w", err)` | `errors.Is` on wrapped |

## 11. Repository Error Handling

The repository is a **translation boundary**. Storage errors in, domain errors out.

```go
// feature/user/repository_impl.go

func (r *repository) FindByID(ctx context.Context, id string) (*User, error) {
    row, err := r.q.GetUser(ctx, id)
    if err != nil {
        if errors.Is(err, sql.ErrNoRows) {
            return nil, &NotFoundError{ID: id}
        }
        return nil, fmt.Errorf("repository.FindByID id=%s: %w", id, err)
    }
    return toModel(row), nil
}

func (r *repository) Save(ctx context.Context, u *User) error {
    _, err := r.q.CreateUser(ctx, toParams(u))
    if err != nil {
        var pgErr *pgconn.PgError
        if errors.As(err, &pgErr) && pgErr.Code == "23505" {
            return ErrEmailTaken
        }
        return fmt.Errorf("repository.Save userID=%s: %w", u.ID, err)
    }
    return nil
}

func (r *repository) Delete(ctx context.Context, id string) error {
    n, err := r.q.DeleteUser(ctx, id)
    if err != nil {
        return fmt.Errorf("repository.Delete id=%s: %w", id, err)
    }
    if n == 0 {
        return &NotFoundError{ID: id}
    }
    return nil
}
```

### Repository Rules

- `sql.ErrNoRows` → always translate to `*NotFoundError` or `ErrNotFound`
- Postgres / Mongo / Redis codes → translate to domain sentinels where applicable
- Unexpected errors → wrap with `fmt.Errorf("repository.Method context: %w", err)`
- Never return raw driver errors
- Never contain business logic

## 12. Usecase Error Handling

The usecase enforces business rules and wraps unexpected errors. Never invents new error types for things already expressed as domain errors.

```go
// feature/user/usecase_impl.go

func (u *usecase) CreateUser(ctx context.Context, input CreateUserInput) (*User, error) {
    // validate first — fail fast before any I/O
    if strings.TrimSpace(input.Name) == "" {
        return nil, &ValidationError{Field: "name", Message: "required"}
    }

    existing, err := u.repo.FindByEmail(ctx, input.Email)
    if err != nil && !errors.Is(err, ErrNotFound) {
        return nil, fmt.Errorf("usecase.CreateUser check email: %w", err)
    }
    if existing != nil {
        return nil, ErrEmailTaken
    }

    newUser := &User{
        ID:        uuid.New().String(),
        Name:      input.Name,
        Email:     input.Email,
        Role:      input.Role,
        CreatedAt: time.Now(),
    }

    if err := u.repo.Save(ctx, newUser); err != nil {
        if errors.Is(err, ErrEmailTaken) {
            return nil, err // race condition — bubble without re-wrapping
        }
        return nil, fmt.Errorf("usecase.CreateUser save: %w", err)
    }

    // side effects only after successful save
    task, _ := NewSendWelcomeEmailTask(SendWelcomeEmailPayload{
        UserID: newUser.ID, Email: newUser.Email,
    })
    _ = u.asynqClient.Enqueue(task)

    return newUser, nil
}

func (u *usecase) DeleteUser(ctx context.Context, id string) error {
    active, err := u.sessionChecker.HasActiveSession(ctx, id)
    if err != nil {
        return fmt.Errorf("usecase.DeleteUser check session: %w", err)
    }
    if active {
        return ErrHasActiveSession
    }

    if err := u.repo.Delete(ctx, id); err != nil {
        var notFound *NotFoundError
        if errors.As(err, &notFound) {
            return err // bubble domain error without re-wrapping
        }
        return fmt.Errorf("usecase.DeleteUser: %w", err)
    }
    return nil
}
```

### Usecase Rules

- Validate input at the top — before any I/O
- Return `*ValidationError` for input failures
- Return sentinels for business rule violations
- Let domain errors bubble from repo without re-wrapping
- Wrap unexpected errors with `fmt.Errorf("usecase.Method context: %w", err)`
- Dispatch side effects only after the main operation succeeds
- Never import `net/http`

## 13. Handler Error Handling

The handler is the **only layer** that maps errors to HTTP status codes. Uses `errors.As` for typed errors, `errors.Is` for sentinels. Unknown errors return 500 and are logged.

```go
// feature/user/handler_http.go

func (h *HTTPHandler) CreateUser(c *gin.Context) {
    var req createUserRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(http.StatusBadRequest, apierror.BadRequest(err.Error()))
        return
    }

    if c.GetString("role") != string(RoleAdmin) {
        c.JSON(http.StatusForbidden, apierror.Forbidden())
        return
    }

    result, err := h.usecase.CreateUser(c.Request.Context(), CreateUserInput{
        Name:  req.Name,
        Email: req.Email,
        Role:  Role(req.Role),
    })
    if err != nil {
        respondError(c, err)
        return
    }

    c.JSON(http.StatusCreated, toResponse(result))
}

// respondError maps domain errors → HTTP responses for this feature.
// errors.As is always checked before errors.Is — typed errors are more specific.
func respondError(c *gin.Context, err error) {
    var valErr *ValidationError
    if errors.As(err, &valErr) {
        c.JSON(http.StatusBadRequest, apierror.BadRequest(valErr.Message))
        return
    }

    var notFound *NotFoundError
    if errors.As(err, &notFound) {
        c.JSON(http.StatusNotFound, apierror.NotFound(notFound.Error()))
        return
    }

    switch {
    case errors.Is(err, ErrEmailTaken):
        c.JSON(http.StatusConflict, apierror.Conflict("email already taken"))
    case errors.Is(err, ErrUnauthorized):
        c.JSON(http.StatusForbidden, apierror.Forbidden())
    case errors.Is(err, ErrHasActiveSession):
        c.JSON(http.StatusConflict, apierror.Conflict("user has an active session"))
    default:
        slog.Error("unhandled error", "error", err, "path", c.Request.URL.Path)
        c.JSON(http.StatusInternalServerError, apierror.Internal())
    }
}
```

### Handler Rules

- `errors.As` before `errors.Is` — typed errors are more specific, checked first
- Never expose internal error messages in 500 responses
- Always log the full error on unexpected cases
- Binding errors handled inline — they never reach `respondError`
- RBAC checked before calling usecase
- Never create new error types in the handler

## 15. Error Flow Summary

```
Repository
  sql.ErrNoRows        →  &NotFoundError{ID}       translate — never let storage errors escape
  pgErr code 23505     →  ErrEmailTaken            translate
  unexpected           →  fmt.Errorf("repo.Method: %w", err)

Usecase
  empty required field →  &ValidationError{Field}  create — typed for field detail
  business rule        →  ErrHasActiveSession       sentinel — known outcome
  domain error         →  return as-is              bubble — do not re-wrap
  unexpected           →  fmt.Errorf("usecase.Method: %w", err)

Handler
  *ValidationError     →  400 BAD_REQUEST           errors.As — checked first
  *NotFoundError       →  404 NOT_FOUND             errors.As — checked first
  ErrEmailTaken        →  409 CONFLICT              errors.Is
  ErrHasActiveSession  →  409 CONFLICT              errors.Is
  ErrUnauthorized      →  403 FORBIDDEN             errors.Is
  anything else        →  500 + slog.Error          default — log full, expose nothing
```

## 16. Rules Cheatsheet

### Testing

```
□ Test files live next to the code they test — never in a /tests subfolder
□ Root-level tests/ only for cross-feature integration and e2e tests
□ Use package user_test (external) — never package user
□ testkit.NewDB()  — shared container, starts once per binary via sync.Once
□ testkit.NewTx()  — per-test transaction, always rolls back, no cleanup needed
□ factory.User / factory.Project — sensible defaults, test sets only what it cares about
□ TestMain in main_test.go for package-level one-time setup (e.g. warm up DB)
□ Repository tests use real DB via testkit — never mock the DB
□ Usecase tests mock Repository and cross-feature interfaces — no DB, no containers
□ Handler tests mock Usecase and use httptest.ResponseRecorder — no DB
□ Each layer tests only its own concerns — no bleed between layers
□ AssertNotCalled for side effects that must not run on error paths
□ AssertExpectations(t) at the end of every mock-based test
□ Mocks live in feature/<n>/mocks/ — never in a central internal/mocks/
□ Mocks generated by mockery — never edited by hand
□ go:generate on interface definitions — regenerate with go generate ./...
□ Commit generated mocks — keeps CI fast, no generation step needed in CI
```

### Error Handling

```
□ Sentinel errors (ErrNotFound) for known outcomes — compared with errors.Is
□ Typed errors (*NotFoundError) when caller needs field detail — compared with errors.As
□ Repository translates ALL storage errors — sql.ErrNoRows never escapes the repo
□ Usecase validates input before any I/O — fail fast with *ValidationError
□ Usecase wraps unexpected: fmt.Errorf("usecase.Method context: %w", err)
□ Usecase lets domain errors bubble without re-wrapping
□ Handler checks errors.As before errors.Is — typed errors are more specific
□ Handler never exposes internal error messages in 500 responses
□ Handler always logs full error on unexpected cases (slog.Error)
```
